Composite Pattern Summary

Use the Composite Pattern when objects form a hierarchical tree structure (parent-child relationships), where:

a single object and
a group of objects

should be treated the same way through a common API.

Best Fit Cases : 
File and Folder systems
Organization charts
Comments and replies
Shape groups
Core Idea

A container object can hold child objects of the same abstraction, and those children can contain more children recursively.

One-Line Rule
Use Composite when:

An object can contain objects of the same abstraction, recursively.

Objects can use other objects via composition
Some composed and singular objects need similar/identical behaviors
Composite design pattern lets us treat both types of objects uniformly
Iteration supported with the Iterator design pattern
