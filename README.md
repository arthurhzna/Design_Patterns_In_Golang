# Design Patterns in Golang

This repository contains small Go examples and notes for learning software design principles and common design patterns.

The material is organized into separate sections so each topic can be studied independently. In general, each section contains short notes and one or more Go examples.

## Purpose

- practice writing Go while learning design patterns
- understand the intent behind each pattern
- compare simple implementations of the same idea
- keep examples small and easy to run

## Repository Overview

This is a study repository focused on:

- software design principles
- creational patterns
- structural patterns
- behavioral patterns
- small parsing and expression examples

## How To Run

Examples in this repository are meant to be run file by file from the repository root.

Use this form:

```bash
go run "path/to/example.go"
```

If a path contains spaces, keep it wrapped in quotes.

## Suggested Study Flow

1. Read the short note for a topic.
2. Run the related Go example.
3. Compare different implementations in the same section.
4. Trace the flow in code and add your own comments if needed.

## Notes

- The code is intentionally simple and learning-oriented.
- Some examples show multiple approaches to the same concept.
- The repository is meant for practice and exploration, not production packaging.
